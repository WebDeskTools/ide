§03 Engine Modules — Active System Components
Astrophysics Module

Nested Noise Fields & Planetary Simulation

Multi-octave Perlin/Simplex noise stacked to simulate realistic planetary crust formation, atmospheric erosion cycles, and large-scale topographic variance.

Hydrology Module

Erosion & Sediment Transport Model

Hydraulic erosion simulation using sinuosity and meander algorithms. Implements a real-time data feedback loop to progressively refine riverbeds and valley carving.

Ecosystem Engine

Species Competition & Biome Definition

Rule-based biome placement driven by elevation and moisture maps. Assets (trees, rocks, shrubs) are placed via spatial competition logic with configurable density parameters.

Planetmics Module

Object Pose & System Toraltion

Manages global coordinate transforms, object poise feedback, and rotation system for planetary-scale terrain alignment and tectonic simulation offsets.

Water Physics

Elevation-Based Sediment Transport

Calculates downstream sediment deposition using elevation gradients. Particles accumulate in depressions and are redistributed dynamically based on slope thresholds.

System Logic

Dynamic Data Flow — Not Static Lines

All module connections operate as live data channels. HTML code interpretation bridges the visual engine to the underlying procedural logic graph in real time.

§05 Common Questions
Where do I get the lesson PDF ticket? +
All lesson PDFs are available in the GitHub repository at webdesktools.github.io/ide/lessons/. Each lesson has its own PDF file. Download the one that matches the module you want to study.
Why does validation fail on my PDF? +
Make sure you are using the original unmodified PDF downloaded directly from the repository. Printing to PDF, re-saving, or compressing the file will destroy the embedded token and invalidate the ticket. Always use the original download.
What 3D printer is required? +
Any FDM (Fused Deposition Modeling) printer works — desktop machines like Bambu Lab, Prusa, Creality, or industrial units. The exported STL is a standard mesh file compatible with all major slicers. A build volume of at least 150×150×50 mm is recommended for terrain models.
Can I save or copy the lesson content? +
The lesson engine runs in a protected mode that disables right-click menus, text selection, and browser developer tools. Only the STL export function is enabled. This protects the course IP while giving you the physical output you need.
Does this work offline? +
PDF validation is fully local — no server call is made. The terrain engine itself is self-contained HTML/JS delivered from the PDF token. Once the lesson tab is open, it runs entirely in your browser without any internet connection needed.
